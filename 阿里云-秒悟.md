## 12. 面试回答稿（问题在上，回答在下）

> 这版改成口语化 + 稍详细，方便你直接说。  
> 每题结构固定：`问题` -> `回答`。

### 12.1 Top20（高频必背）

**1）LLM 和 Agent 的本质区别是什么？**  
回答：你可以这么说：LLM本质上是一次性生成器，给它一个输入，它返回一个输出；Agent是“会自己循环做事的系统”，通常包括模型、工具、记忆、规划和执行控制。简单点讲，LLM更像会思考，Agent是“会思考+会动手+会复盘”。

**2）ReAct 循环怎么实现？怎么避免死循环？**  
回答：ReAct就是“先想一下，再调工具，再看结果，再继续想”，一直循环到完成。工程上我会加三个保险：第一，最大步数限制；第二，重复动作检测；第三，低收益熔断，比如连续两轮没有新信息就主动停下来或降级。

**3）Function Calling 的原理是什么？LLM 真执行函数吗？**  
回答：LLM不执行函数，它只负责输出结构化调用意图，比如函数名和参数。真正执行是业务代码在做，执行完再把结果回填给模型。这个分层很关键，因为权限控制和审计都在业务层。

**4）MCP 是什么，解决了什么问题？和 Function Calling 的关系？**  
回答：Function Calling解决“怎么调用一个函数”，MCP更像“工具接入标准”，解决多工具、多团队协作时的协议统一问题。我的理解是：Function Calling是调用格式，MCP是生态层标准化。

**5）你做过的 Agent 系统架构是什么？核心模块如何分层？**  
回答：我一般分六层：Orchestrator（调度）、Planner（规划）、Tool Layer（工具执行）、Memory（短期+长期）、Guardrails（安全治理）、Observability（监控评估）。这样做的好处是，每层职责清楚，后续扩展模型或工具不会影响整体架构。

**6）Agent 的短期记忆和长期记忆怎么设计？**  
回答：短期记忆用于当前任务上下文，通常放会话窗口或缓存；长期记忆放向量库和结构化库，存历史结论、偏好和业务知识。为了防止上下文爆炸，我会做摘要压缩，把长会话变成结构化摘要再回灌。

**7）RAG 完整链路（离线/在线）怎么设计？**  
回答：离线侧做数据清洗、切块、嵌入、建索引；在线侧做query改写、召回、重排、上下文拼装、答案生成和引用。我的经验是，离线质量决定上限，在线策略决定稳定性和实时体验。

**8）Chunk size / overlap 你如何选？线上怎么调参？**  
回答：我会先按语义边界切块，不盲目按固定长度。初始参数一般从中等块长+小重叠开始，然后看Recall@k、答案正确率和延迟指标迭代。目标是“召回准”同时“上下文不过载”。

**9）如何提升 RAG 检索质量（Hybrid、Rerank、Query Rewrite）？**  
回答：我通常是三段式：先做query rewrite让问题更可检索，再做hybrid召回提升覆盖，最后用cross-encoder重排提精度。尤其业务词和缩写多的场景，hybrid效果很明显。

**10）如何评估 RAG（检索指标 + 生成指标）？**  
回答：检索层看Recall@k、MRR、NDCG；生成层看Faithfulness、Relevance和引用覆盖率。线上还会加人工抽检，特别是高风险场景，避免只看自动分数。

**11）如何做 Prompt 版本管理和 A/B 实验？**  
回答：我把Prompt当代码管理，每次变更要有版本号、变更原因和回滚点。A/B时固定评测集和统一评估器，先离线通过再灰度上线，避免“换了提示词但不知道好坏”。

**12）LangGraph 的 State、Node、Edge、Conditional Edge 分别是什么？**  
回答：State是共享上下文，Node是执行步骤，Edge是固定流转，Conditional Edge是按状态动态路由。你可以理解成“带状态的流程图引擎”，适合复杂可回溯任务。

**13）Supervisor 与 Swarm / Hierarchical 多智能体模式怎么选？**  
回答：如果场景复杂且强调可控性，我优先Supervisor；如果追求速度且任务比较标准化，可以考虑Swarm；如果任务是多层分工，就用Hierarchical。核心取舍是：可控性、延迟、误路由风险。

**14）Agent 的可观测性你看哪些指标？**  
回答：我看六类：质量分、延迟、token与成本、错误率、工具调用成功率、降级/回退率。再加trace把每一步串起来，这样出了问题可以快速定位是检索、工具还是生成环节。

**15）线上如何控制 Agent 成本与延迟？**  
回答：三件事最有效：模型分级（高价值请求用强模型）、缓存（检索和答案层）、限制循环步数。长任务异步化也很关键，避免请求链路被拖慢。

**16）如何防 Prompt Injection、工具滥用和越权调用？**  
回答：我会分三层防护：输入层做注入检测和清洗；工具层做白名单、参数校验、最小权限；执行层对高风险写操作加人工审批。安全一定是体系化，不是靠一条提示词。

**17）Python 并发里 threading / multiprocessing / asyncio 如何选？**  
回答：I/O密集优先asyncio，CPU密集用multiprocessing，threading主要用于兼容阻塞库或过渡方案。核心原则就是：按瓶颈选模型，不要一把梭。

**18）高并发下缓存、消息队列、微服务如何组合？**  
回答：我的常见组合是：网关限流 -> Redis缓存 -> MQ削峰 -> 异步Worker处理重任务 -> DB读写分离。这样可以把瞬时流量和慢任务隔离开，系统更稳。

**19）你如何从 0 到 1 推一个 AI 功能上线并持续优化？**  
回答：我会先定义MVP和可量化指标，然后小流量试点，稳定后灰度扩大。上线后用监控和评估回路持续迭代，不是“上线就结束”，而是“上线才开始优化”。

**20）你做过最失败的一次 AI 落地是什么，如何复盘？**  
回答：建议你讲一个“检索误召回或工具误调用导致回答错误”的真实例子。重点说三件事：怎么止损、怎么复盘、怎么机制化（比如加评估门禁、fallback和监控报警），这样面试官会觉得你有闭环能力。

### 12.2 其他题目怎么回答（通用模板）

如果面试官问到第2到第10章里的任意题目，你都可以按下面这个模板答，基本不会乱：

1. **先定义**：这个概念是什么，和相邻概念区别是什么。  
2. **再讲方案**：你实际怎么做，流程是什么。  
3. **讲取舍**：为什么这么选，不这么选会有什么问题。  
4. **落指标**：你用什么指标判断它有效。

示例（可套任何题）：

> 这个问题我通常分两层看：先保证可用，再保证可优化。  
> 方案上我会先做A，再做B，最后做C。  
> 取舍点主要是质量、延迟和成本。  
> 最后我会用【指标1】【指标2】来验证效果，达不到阈值就回滚或降级。

## 13. Top20 超口语版（每题两句话）

1）**LLM 和 Agent 区别**  
LLM更像一次性回答器，给输入吐输出。Agent是会“想、做、看结果、再调整”的闭环系统。  

2）**ReAct 怎么防死循环**  
ReAct就是思考、行动、观察循环。工程上加步数上限、重复动作检测和低收益熔断就能防大多数死循环。  

3）**Function Calling 原理**  
模型只负责说“我要调哪个函数、参数是什么”。真正执行在你的代码里，执行结果再喂回模型。  

4）**MCP 是啥**  
Function Calling是调用格式，MCP是工具接入标准。MCP的价值是多工具统一协议，系统更好扩展。  

5）**Agent 架构怎么分层**  
我一般分调度、规划、工具、记忆、安全、观测六层。这样每层职责清晰，后续换模型或换工具不容易炸。  

6）**记忆怎么做**  
短期记忆管当前任务上下文，长期记忆放向量库和结构化库。会话太长就做摘要压缩，不然上下文会爆。  

7）**RAG 链路**  
离线先清洗切块建索引，在线再检索重排拼上下文生成答案。离线决定上限，在线决定稳定性。  

8）**Chunk 怎么调**  
先按语义切，再按指标调大小和重叠。看Recall@k、答案正确率和延迟，三者平衡就行。  

9）**怎么提检索质量**  
常规是Query Rewrite + Hybrid召回 + Cross-Encoder重排。先保召回，再提精度。  

10）**RAG 怎么评估**  
检索看Recall@k、MRR、NDCG。生成看Faithfulness、相关性和引用覆盖率。  

11）**Prompt 版本与A/B**  
Prompt要像代码一样版本化，能追溯能回滚。A/B用固定数据集跑，达标再灰度上线。  

12）**LangGraph 核心概念**  
State是共享状态，Node是步骤，Edge是流转，Conditional Edge是动态路由。它本质是可恢复的状态机编排。  

13）**Supervisor / Swarm 怎么选**  
要稳和可控就Supervisor。要快且任务标准化就Swarm，复杂分层任务用Hierarchical。  

14）**可观测性看什么**  
我重点看质量、延迟、成本、错误率、工具成功率。再配trace把问题定位到具体步骤。  

15）**成本和延迟怎么控**  
模型分级、缓存、减少无效检索、限制循环步数最有效。长任务异步化能显著降接口时延。  

16）**安全怎么做**  
输入做注入检测，工具做白名单和参数校验，执行层做最小权限。高风险写操作必须人工审批。  

17）**Python 并发怎么选**  
I/O密集用asyncio，CPU密集用multiprocessing。阻塞库短期可线程池兜底，长期改异步。  

18）**高并发后端组合**  
网关限流、Redis缓存、MQ削峰、异步Worker、DB读写分离。核心是把慢任务和主链路隔离。  

19）**0到1上线怎么做**  
先做MVP和指标，再小流量试点，最后灰度放量。上线后持续用监控和评估驱动优化。  

20）**失败复盘怎么讲**  
讲一次真实翻车，先说怎么止损，再说怎么复盘，最后说机制怎么补。面试官最看重的是你有没有闭环。
